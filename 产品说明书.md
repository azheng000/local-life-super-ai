## 一、产品概述

### 1.1 产品定位

**本地生活超级AI** 是一个 AI 驱动的本地生活服务平台，让用户通过自然语言对话即可完成所有本地生活服务需求——餐饮、外卖、酒店、娱乐、旅游、用车等。

**核心创新**：
- 用户端：一个入口，对话即服务（类似豆包的交互体验）
- 商家端：每个商家拥有独立智能体，7×24小时秒级响应

### 1.2 解决什么问题？

| 用户痛点 | 我们的解决方案 |
|----------|----------------|
| 打开N个APP比价 | 一句话描述需求，AI自动匹配最优 |
| 商家响应慢（平均2小时） | 智能体秒级响应，即时确认预约 |
| 选择困难症 | 基于人情味数据，推荐真正适合的 |
| 没人情的推荐 | 商家故事、老板性格、真实评价 |

| 商家痛点 | 我们的解决方案 |
|----------|----------------|
| 平台抽成20-30% | 订阅制，商家100%收入 |
| 付费推广才曝光 | AI匹配，好服务自然被发现 |
| 客服成本高 | 智能体7×24小时自助服务 |
| 数据不在自己手里 | 商家拥有完整客户数据资产 |

### 3.2 匹配逻辑

```javascript
// 匹配算法示意
function matchAgent(userQuery, userLocation, userPreference) {
  
  // 第一步：理解用户意图
  const intent = analyzeIntent(userQuery)
  // { category: "餐厅", subCategory: "海鲜", budget: 300, atmosphere: "浪漫" }
  
  // 第二步：召回候选商家
  const candidates = await db.query(`
    SELECT * FROM agents 
    WHERE category = '餐厅'
    AND ST_Distance(location, $1) < 5000  -- 5公里内
    AND status = 'active'
    ORDER BY trust_score DESC
    LIMIT 20
  `, [userLocation])
  
  // 第三步：计算匹配分数
  const scored = candidates.map(agent => ({
    ...agent,
    matchScore: calculateMatch(agent, intent, userPreference)
  }))
  
  // 第四步：返回Top3
  return scored.sort((a, b) => b.matchScore - a.matchScore).slice(0, 3)
}

function calculateMatch(agent, intent, preference) {
  let score = 0
  
  // 类目匹配（40%）
  if (agent.category === intent.category) score += 40
  
  // 价格区间匹配（20%）
  if (isPriceMatch(agent.priceRange, intent.budget)) score += 20
  
  // 信任分（20%）
  score += agent.trustScore * 0.2
  
  // 距离权重（10%）
  score += (1 - distance / 5000) * 10
  
  // 人情味加分（10%）
  score += agent.storyBonus * 0.1
  
  return score
}
```

```sql
-- 1. 智能体主表
CREATE TABLE agents (
  id SERIAL PRIMARY KEY,
  agent_id VARCHAR(50) UNIQUE NOT NULL,  -- AGT_1713123456_ABCD1234
  agent_type VARCHAR(20),                 -- merchant/individual
  status VARCHAR(20) DEFAULT 'active',

-- 2. 基础信息表
CREATE TABLE agent_basic_info (
  agent_id VARCHAR(50) PRIMARY KEY,
  name VARCHAR(100),
  category VARCHAR(50),          -- 餐厅/酒店/娱乐
  address TEXT,
  location GEOGRAPHY(POINT),     -- PostGIS空间字段
  contact VARCHAR(50),
  owner_name VARCHAR(50)
);

-- 3. 品牌信息表（人情味数据）
CREATE TABLE agent_brand_info (
  agent_id VARCHAR(50) PRIMARY KEY,
  story TEXT,                    -- 商家故事
  personality VARCHAR(50),       -- warm_and_friendly/professional/humorous
  keywords TEXT[],               -- ['海鲜', '新鲜', '实惠']
  created_at TIMESTAMP DEFAULT NOW()
);

-- 4. 服务信息表
CREATE TABLE agent_service_info (
  agent_id VARCHAR(50) PRIMARY KEY,
  services JSONB,                -- [{name, price, description}]
  price_range NUMERIC[],         -- [50, 300]
  business_hours VARCHAR(50),    -- "09:00-21:00"
  special_tags TEXT[]            -- ['包厢', '停车位', 'WIFI']
);

-- 5. MCP配置表
CREATE TABLE agent_mcp_config (
  agent_id VARCHAR(50) PRIMARY KEY,
  endpoint VARCHAR(255),         -- MCP Server URL
  tools JSONB,                   -- 可用工具列表
  status VARCHAR(20) DEFAULT 'active'
);

-- 6. 信任分表
CREATE TABLE agent_trust_score (
  agent_id VARCHAR(50) PRIMARY KEY,
  overall_score NUMERIC DEFAULT 5.0,
  response_rate NUMERIC,         -- 响应率
  completion_rate NUMERIC,       -- 完单率
  avg_rating NUMERIC,
  total_reviews INT DEFAULT 0
);

-- 7. 预约订单表
CREATE TABLE reservations (
  id SERIAL PRIMARY KEY,
  reservation_id VARCHAR(50) UNIQUE,
  agent_id VARCHAR(50),
  customer_phone VARCHAR(20),
  customer_name VARCHAR(50),
  reservation_date DATE,
  reservation_time TIME,
  party_size INT,
  status VARCHAR(20),            -- pending/confirmed/completed/cancelled
  created_at TIMESTAMP DEFAULT NOW()
);

-- 8. 评价表
CREATE TABLE reviews (
  id SERIAL PRIMARY KEY,
  agent_id VARCHAR(50),
  reservation_id VARCHAR(50),
  rating INT,                    -- 1-5
  content TEXT,
  is_verified BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 9. 用户偏好表
CREATE TABLE user_preferences (
  user_id VARCHAR(50) PRIMARY KEY,
  preferred_categories TEXT[],
  budget_range NUMERIC[],
  dietary_restrictions TEXT[],
  interaction_history JSONB
);
```

### 5.2 商业模式

#### 收入来源
```
1. 商家订阅费（主要）
   - 基础版：299元/月
   - 专业版：599元/月（含数据分析）
   - 旗舰版：999元/月（含营销工具）

2. 增值服务
   - 智能体定制：一次性2000-5000元
   - 数据洞察报告：按需付费
   - 营销活动策划：按效果付费

3. 未来扩展
   - 供应链金融服务
   - 商家培训课程
   - 行业数据API
```

#### 为什么不抽佣金？
```
抽佣模式的问题：
- 商家反感，会引导线下交易
- 利益对立，商家想提高客单价，平台想提高GMV
- 数据不真实，商家隐藏真实价格

订阅模式的优势：
- 利益一致，商家赚得越多越愿意续费
- 数据真实，商家愿意公开真实数据
- 关系稳固，长期合作积累信任
```

### 5.4 抖音优先策略

#### 为什么选择抖音而非美团？

```
决策维度评分（满分10分）：

| 维度 | 抖音 | 美团 | 权重 |
|------|------|------|------|
| 缺陷刚需度 | 9分（核销率是生存问题） | 7分（AI无人情味是体验问题） | 30% |
| 技能匹配度 | 9分（智能体解决核销） | 6分（需要长期数据积累） | 25% |
| 增量价值 | 9分（核销率提升=1700亿） | 7分（效率提升=数百亿） | 25% |
| 反噬风险 | 7分（内容基因难内化履约） | 3分（易被复制边缘化） | 20% |

加权得分：
- 抖音：8.7分
- 美团：6.4分
```

#### 抖音合作切入点
```
抖音痛点：核销率仅50%，买了不用是最大浪费

我们的解决方案：
1. 商家智能体即时响应 → 提升用户决策效率
2. 预约确认机制 → 减少无效下单
3. 消费提醒服务 → 提升到店转化率

预期效果：
- 核销率从50%提升到70%+
- 抖音本地生活GMV真实转化翻倍
```
